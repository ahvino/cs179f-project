rmdir and my_dir

By JS, GBF, MT, CK

Completed implementation of rmdir and created the my_dir struct. The struct had a mode, access count, flags and an offset. As well, we had a pointer to our directory of inodes which was also a struct. rmdir had starts with all our error checking i.e. (is path root, DNE). We then call our split function to get the appropriate paths. We then find the location of the dirent and we remove the dirent from the parent followed by erasing the inode from the list.

Updated the File System structure

By JS, GBF, MT, CK

Our file system is mainly comprised of a struct dirent_frame, class file and class ilist. The dirent frame is is our type used in our vector of directory entries. Our class file just holds all the data for a file and our ilist is our list where we map our entries with the ino_t being the key and the File as the value.

getting the inode

By JS, GBF, MT, CK

We implemented the find_ino function. Basically we take in a string path and check this path against S_ISLINK. we use readlink in our check and then call our find_real_ino function. We use lookup(it, fh) to find the actual inode file handler.
